Terminal Commands

  • pwd – Show current directory structure.
  • date – Show current date on system.
  • cd | ../ | - | ../../ – Move through folders. Move 1 folder back. Move 2 folders back.
  • ls | -a | -al – List Directories. List All including hidden. List All Files with file permissions.
  • start . – Open GUI of Folder.
  • start – Open GUI file.
  • curl -0 – Saves a file from the web to your local folder you are in.
  • echo – Output on the next line.
  • history – See history.
  • touch – Create new file.
  • mkdir – Create directory
  • rm – Delete file.
  • rmdir – Delete Empty Directory Only
  • rm -r | -rf – Delete Directory and all its content including files Force Delete Directory and all its content including files. (Never use -rf unless you have to)
  • cp – Copy file.
  • cp -r – Copy directory content to new location.
  • mv – Move Items Files and Directories
  • unzip – Unzip file.
  • cat – Prints to screen what is in the file.
  • clear – Clear screen.
  • exit – Exit Terminal.
  • q – Exit file.
  • ping 10.1.1.1 – Check to see if you are connected and your computer is pinging to the internet.

Git Stuff

  • HOME= – Choose home directory for a git session.
  • git config --list – List your Git configuration.
  • git init –Start your git repository and start tracking files. This will create a .git folder in your current directory.
  • git clone – Downloading the latest version of a remote project repository.
  • git fetch – get all the updates from the remote repository, including new branches.
  • git branch – get all the updates from the remote repository, including new branches.
  • git branch | – Create new branch. Creates a branch from a past commit.
  • git branch | -a – If you want to see your local and remote branches.
  • git branch -d | -D – Delete branch. Force delete will delete unsaved work in branch.
  • git log | --oneline – To see commit details or to see one line preview of commits.
  • git log --stat | --stat– To see commit in more detail including author, date and files worked on. (You can get the sha when you run the previous command)
  • git log --graph – To see tree structure of branch and commits.
  • git log --graph --decorate --oneline – A easy way to see all commits and line tree and tags.
  • git log -p – Shows full changes to files. This is a Patch output.
  • git log -n [numbercount] – Show logs limited to number count.
  • git reflog – List operations (e.g. checkouts or commits) made on local repository.
  • git push --set-upstream origin – If the branch is not yet tracked, and only resides on the local machine.
  • git add --all | -A | . – Add new files. This is called staging.
  • git reset – Remove files and move files back to unstaged.
  • git commit -m | --amend -m – Commit changes to repository. Amend previous commit message.
  • git commit -a -m – Bypass staging and just commit message. Stage and commit together.
  • git revert – Reverts your commit from sha. You can see file in earlier state from that commit point.
  • git status – Check to see if there are any staged files. If in red you need to stage them.
  • git push – Push commits to remote repository.
  • git push -u origin – Push your local branch to remote repository with upstearm.
  • git remote -v – Show the associated remote repositories and their stored name and show links to branch.
  • git remote show origin – Show if your remote branches are tracked. Get more info about the remote repository.
  • git push -d origin – Delete a remote branch.
  • git pull – Update your local working branch with commits from the remote and update all remote tracking branch and apply changes to local branch.
  • git fetch – Pulls changes from remote but will not apply it to your local branch. Use “git pull” for that.
  • git stash | --all – Stash away unstaged items. This helps you jump between branches.
  • git stash apply | pop – Gets unsaved documents and keeps it ready for staging.
  • git stash drop – Delete stashed items.
  • git checkout – Checkout from current branch and go to another branch.
  • git checkout -b – Checkout from current branch and create a new branch.
  • git diff | --staged – See changes made between each other unstaged or staged files.
  • git merge – Merge from another local branch.
  • git rebase – Brings your current branch up to date with a branch already ahead of your branch.
  • git tag | - d – Add tag. Delete tag. Tags are attached to your commits.
  • git tag -a – Add a tag to a commit already made.
